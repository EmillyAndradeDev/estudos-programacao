Introdução ao HTML

Conceitos Básicos - tags <head></head>

  • A tag <html lang="pt-br"> indica a língua do meu site.
  • O que vem dentro da tag <head></head> não aparece no meu site, são informações apenas para os devs e os navegadores. Essa é a parte de configurações.
  • A tag <meta charset="UTF-8"> indica que o meu site segue um padrão de codificação de caracteres que suporta acentuação.
  • A tag <meta name="viewport" content="widht=device-width, initial-scale=1.0"> indica que meu site vai ocupar 100% da tela do navegador do usuário.
  • A tag <title></title> é o título que aparece na aba do navegador.
  • Tudo o que vem dentro da tag <body></body> é o corpo do meu site.
  • A tag <link rel="shortcut icon" href="img/favicon.ico" type="image/x-icon"> é o ícone do navegador do meu site. A imagem para essa tag deve estar dentro da pasta do projeto e deve estar no formato ico.
  • HTML significa: Hiper Text Markup Language, ou seja, texto de marcação. Sendo assim, as tags podem ser chamadas de marcas.

Parágrafos

  • Você pode escrever um parágrado colocando seu texto entre as tags <p></p>.
  • Cada parágrafo deve ser iniciado dentro de uma tag <p></p>.
  • A quantidade de "enter" que eu dou no código não interfere para pular linhas ou iniciar um novo parágrafo. O que importa aqui é a tag(marca).

Quebras de Linha

  • Se precisar quebrar o texto em algum lugar específico, é preciso usar a tag <br>.
  • Não coloque mais de uma tag <br> para criar espaçamento no seu código. Prefira sempre o CSS para dar espaços maiores (é mais profissional).

Tag de Imagem

Imagens Internas

    A tag padrão de imagem é essa: <img src="" alt="">
  • Para imagens que estão na pasta principal do nosso projeto, usamos dentro do parâmetro src="" apenas o nome da imagem. Exemplo: <img src="minhaimagem.png" alt="minha imagem">
  • Para imagens que estão em outra pasta dentro do nosso projeto, usamos dentro do parâmetro src="" NomeDaPasta/NomeDaImagem. Exemplo: <img src="minhapasta/minhaimagem.png" alt="minha imagem">

Imagens Externas

  • Para imagens de outras fontes, basta escolher a imagem e copiar o url do endereço da imagem.
  • Nesse caso usamos o link dentro do parâmetro src="". Exemplo: <img src="https://cdn.iconscout.com/icon/free/png-256/free-javascript-logo-icon-download-in-svg-png-gif-file-formats--technology-social-media-company-vol-4-pack-logos-icons-3031259.png" alt="Logo JavaScript">
  • Esse método não é muito indicado pois a imagem está em outro servidor, e caso esse servidor saia do ar, a imagem vai sumir do meu site.

Favicon

  • A tag do favicon fica dentro da tag <head></head> localizada acima da tag <title></title>.
  • Essa é a tag favicon: <link rel="shortcut icon" href="img/favicon.ico" type="image/x-icon"> . Quando digitamos link dentro da tag <head></head>, o VSCode sugere link:favicon, basta selecionar e a tag será completada automaticamente.
  • Após a sugestão do VSCode, devemos alterar apenas o que vem dentro do href="" que é a localização da nossa imagem de favicon dentro do nosso projeto. Ou seja, pasta/nomeDoArquivo.ico
  • A imagem do favicon precisa estar no formato .ico
  • Podemos transformar uma imagem no formato png em .ico usando o site favicon.io
  • As imagens no formato .ico ficam marcadas dentro do VSCode com uma estrela.

Formatação de textos - Negrito e Itálico

Negrito / Destaque

Nesta frase, temos um termo em negrito usando a tag <b></b>. Essa tag é uma tag não semântica. É uma tag de forma.

Nesta frase, temos um termo em destaque usando a tag <strong></strong>. Essa tag é uma tag semântica. Ela tem significado e forma.

  • A tag <b></b> só tem forma. Enquanto a tag <strong></strong> tem forma e sentido.
  • A tag <strong></strong> diz que coloquei o termo dentro da tag porque ele é um termo strong/termo forte/termo de destaque.

Itálico / Ênfase

Nesta frase temos um termo em itálico usando a tag <i></i>. Essa tag é não semântica, ela tem apenas a forma itálica.

Nesta frase, temos um termo em destaque em ênfase usando a tag <em></em>. Essa tag é semântica.

  • Visualmente, as duas tags funcionam. Porém a tag <em></em> tem além de forma um significado, e por ter significado ela é considerada uma tag semântica. Ou seja, a tag <em></em> significa que o texto tem ênfase.

Recomendação do w3c:

  • HTML precisa ter: sentido, pode ou não ter forma. Foca em tags semânticas.
  • CSS é feito para ter forma. As tags não são semânticas.

Texto marcado (marcador de texto):

Podemos criar também um texto marcado usando a tag <mark></mark>

Em html não temos como mudar a cor. Quando uso a tag <mark></mark> eu estou dizendo para o meu navegador que quero o texto marcado, isso é semântico.

A cor da marcação dessa tag deve ser alterada com o CSS dentro da própria tag usando a tag <style></style>. Exemplo de texto marcado com css: marcado de cor diferente. Isso é estilo/forma.

E neste outro parágrafo, temos outro texto marcado apenas com tag semântica, sem herdar a característica da cor da tag <mark></mark> do parágrafo acima.

Definindo a mesma cor na tag <mark></mark> para todos os itens que estiverem dentro da tag <mark></mark> nessa página:

É necessário dentro da tag <head></head>, logo após a tag <title></title> abrir uma tag <style></style> e usar o seletor mark seguido dos bigodinhos.

Dentro dos bigodinhos eu coloco a característica que eu quero na tag <mark></mark>, que no caso é a background-color: seguida da cor(nome ou código) e ponto e vírgula no final.

Como no exemplo de código abaixo:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="shortcut icon" href="img/favicon.ico" type="image/x-icon">
    <title>Formatação de textos - Negrito e Itálico</title>
    <style>
        mark {
            background-color: darkcyan;
        }
    </style>
</head>

Esse exemplo, se posicionado no lugar certo, faz com que todos os itens de tag <mark></mark> tenham sua cor alterada para a cor definida dentro da tag style na head.

Texto grande e pequeno

Estamos criando um texto grande e um texto pequeno nesse parágrafo.

A tag <big></big> pode ficar na cor vermelha dentro do VSCode porque já está obsoleta dentro do html5.

Por que a tag <big></big> ficou obsoleta? Porque ao aumentar uma letra, não significa que você deu um significado para ela. Você está apenas dando forma. E forma deve ser sempre no CSS.

Por que a tag <small></small> não ficou obsoleta? Porque small também é significado além de ser forma. Como por exemplo em contratos usamos "leia as letras pequenas". Letras pequenas têm um sentido para estar ali. É um texto meio que secundário, mas que também deve ser lido.

Texto excluído / deletado

Podemos marcar um texto como excluído para indicar que ele deve ser lido, mas não considerado. (tag semântica, tem sentido)

Nesse caso, usamos a tag <del></del>

Texto inserido

Podemos marcar um texto como inserido para dar uma ênfase e indicar que ele foi adicionado depois. (tag semântica, tem sentido)

Nesse caso, usamos a tag <ins></ins>

O texto inserido fica sublinhado.

Existe também o texto sublinhado, com a tag <u></u> que é uma tag não semântica, porque ela só diz que o texto foi sublinhado, mas não diz o sentido de estar sublinhado.

Texto sobrescrito

Usado para inserir coisas do tipo x2+3 (ou seja, x ao quadrado mais 3)

Nesse caso, usamos a tag <sup></sup>

Texto subescrito

Usado para inserir coisas do tipo: H2O (fórmula da água)

Nesse caso, usamos a tag <sub></sub>
