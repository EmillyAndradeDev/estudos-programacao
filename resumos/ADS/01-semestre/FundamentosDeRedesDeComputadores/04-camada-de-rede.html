Camada de Rede

Objetivos

  • Reconhecer os conceitos básicos da camada de rede no Modelo OSI.
  • Aplicar configurações de rede IP.
  • Classificar os protocolos de controle da Internet.
  • Selecionar os algoritmos de roteamento.

Conceitos Básicos da Camada de Rede (OSI)

  • A camada de rede é responsável por endereço lógico, roteamento e encaminhamento de pacotes entre redes diferentes.
  • Comutação de pacotes (store-and-forward): cada nó recebe, armazena e encaminha pacotes.
  • Circuitos virtuais vs Datagramas:
    • Circuito virtual → caminho pré-estabelecido; pacotes seguem na mesma ordem.
    • Datagrama → cada pacote pode seguir um caminho diferente; entrega não garante ordem.
  • Endereçamento na camada de rede:
    • Hierárquico → divide rede em sub-redes e hosts.
    • Horizontal → dentro da mesma rede local.
    • Mapeamento direto e vinculação dinâmica → relaciona endereços lógicos (IP) com físicos (MAC).
  • ARP (Address Resolution Protocol) → tradutor de IP para MAC dentro da rede local.

→ Exemplo prático:

    Quando você acessa um site:
  1. O seu computador usa o IP do servidor para roteamento.
  2. O ARP traduz o IP do roteador local para seu endereço MAC, permitindo o envio correto do pacote.

Configurações de Rede IP

  • IPv4: protocolo mais usado na internet; define endereços de 32 bits.
    • Endereço IPv4: 32 bits, dividido em rede e host.
  • Fragmentação: pacotes grandes podem ser divididos para atravessar redes com limites de tamanho.
  • Classes de IP (A, B, C): definem quantidade de redes e hosts.
    • Classe A: muitas máquinas por rede.
    • Classe B: redes médias.
    • Classe C: poucas máquinas por rede.
  • Sub-redes: divisão da rede para melhor organização e segurança.
  • Máscara de rede: identifica a parte de rede e host do endereço IP.
  • NAT (Network Address Translation): permite que múltiplos dispositivos usem um único IP público.
    • Traduz IPs privados em IPs públicos, permitindo que várias máquinas usem um único endereço público.
    • Importante para economizar endereços IPv4 e aumentar a segurança.

→ Exemplo prático:

Seu roteador doméstico usa NAT para que todos os dispositivos (celular, notebook, TV) acessem a internet usando um único IP público.

Protocolos de Controle

  • DHCP (Dynamic Host Configuration Protocol)
    • Configura IP, máscara, gateway e DNS automaticamente.
    • Tipos de atribuição: manual, automática e dinâmica.
    • Estados do cliente: Descoberta, Oferta, Requisição, Confirmação, Renovação, Liberação.
  • ICMP (Internet Control Message Protocol)
    • Usado para enviar mensagens de erro ou teste de conectividade.
    • Exemplo: comando ping utiliza ICMP para testar se um host está ativo.
  • ARP
    • Traduz endereços IP em endereços MAC para entrega local de pacotes. (Descobre o endereço físico (MAC) correspondente a um IP)

→ Exemplo prático:

Quando seu computador obtém IP automaticamente via DHCP ao se conectar ao Wi-Fi, ele está usando a camada de rede para configurar corretamente a comunicação.

Algoritmos de Roteamento

  • Objetivo do roteamento → encontrar o melhor caminho para os pacotes.
  • Princípio da otimização → escolher caminhos eficientes e confiáveis.
  • Roteamento pelo caminho mais curto (Dijkstra) → calcula menor custo entre dois pontos.

Tipos de roteamento:

  • Não adaptativos → tabelas de roteamento fixas.
  • Adaptativos → ajustam-se dinamicamente às mudanças da rede.
    • Globais → conhecem toda a topologia da rede (ex: Dijkstra - caminho mais curto).
    • Descentralizados → roteadores conhecem apenas vizinhos (ex: vetor de distâncias - Bellman-Ford).
  • Roteamento interno (IGP) → dentro de uma mesma rede ou organização.
  • Roteamento externo (EGP) → entre diferentes redes/organizações.
  • Roteamento de vetor de distâncias → cada roteador conhece distância para outras redes e compartilha com vizinhos.
  • Roteamento de estado de enlace → cada roteador conhece toda a topologia da rede e calcula caminhos.
  • Roteamento hierárquico → divide grandes redes em regiões para simplificar o roteamento.

→ Exemplo prático:

Quando você acessa Netflix, seu pacote percorre múltiplos roteadores. Cada roteador decide o próximo salto usando algoritmos de roteamento adaptativos para otimizar tempo e evitar congestionamento.

A camada de rede garante que os dados sejam enviados corretamente de um host a outro em diferentes redes, usando endereçamento lógico (IP), protocolos de controle (DHCP, ARP, ICMP) e algoritmos de roteamento eficientes. É a camada que conecta diferentes redes, permitindo que a internet funcione globalmente.

→ Exemplo prático:

    Ao acessar www.youtube.com:
  1. O navegador envia uma solicitação HTTP (camada de aplicação).
  2. O DNS traduz o nome para o IP do servidor.
  3. O IP do servidor e do cliente são usados pela camada de rede.
  4. O roteador usa algoritmos de roteamento (Dijkstra ou vetor de distâncias) para escolher o melhor caminho.
  5. Se necessário, o NAT traduz o IP privado do seu dispositivo em IP público.
  6. Os pacotes são entregues ao destino e o vídeo é exibido.
