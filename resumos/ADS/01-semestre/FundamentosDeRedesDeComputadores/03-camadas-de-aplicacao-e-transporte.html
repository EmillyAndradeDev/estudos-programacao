Camadas de Aplicação e Transporte

Objetivos

  • Reconhecer as arquiteturas de aplicações.
  • Identificar os principais serviços oferecidos pela camada de aplicação.
  • Localizar os elementos da camada de transporte.
  • Comparar os serviços oferecidos pela camada de transporte.

Arquiteturas e Camada de Aplicação

  1. Camada de Aplicação
    • É onde os usuários e processos interagem diretamente com a rede.
    • Fornece serviços para troca de dados entre aplicações distribuídas
  2. Arquiteturas de Aplicações
    • Cliente-Servidor:
      • O cliente faz requisições.
      • O servidor fornece respostas.
      • Exemplo: Navegador (cliente) e YouTube/Google (servidor).
    • Peer-to-Peer (P2P):
      • Todos os nós podem atuar como clientes e servidores.
      • Exemplo: redes de compartilhamento de arquivos (BitTorrent).

Serviços da Camada de Aplicação

  • Protocolos mais importantes:
    • HTTP/HTTPS → Web (navegação, vídeos, sites).
    • SMTP, POP3, IMAP → Correio eletrônico.
    • DNS → Serviço de nomes (tradução de nomes de domínio em endereços IP).
  • Funcionamento do Serviço Web (HTTP)
    • Cliente envia requisição ao servidor (GET/POST).
    • Servidor responde com código de status (200 OK, 404 Not Found).
    • O navegador interpreta e exibe o conteúdo.
  • Correio eletrônico
    • SMTP (Simple Mail Transfer Protocol): envia mensagens entre servidores.
    • POP3 (Post Office Protocol): baixa mensagens para o cliente (remove do servidor).
    • IMAP (Internet Message Access Protocol): acessa mensagens diretamente no servidor (sincronização).
  • Serviço de Nomes - DNS
    • Converte nomes (www.exemplo.com) em endereços IP.
    • Funciona como uma "agenda telefônica da Internet".
    • Utiliza servidores distribuídos hierarquicamente.

Elementos da Camada de Transporte

  1. Funções principais:
    • Garante comunicação processo a processo (não apenas máquina a máquina).
    • Faz a multiplexação/demultiplexação:
      • Multiplexação → Vários processos usam a mesma conexão.
      • Demultiplexação → Identificação correta do processo de destino por meio das portas.
  2. Endereçamento (Portas):
    • Cada processo tem um número de porta único.
    • Exemplos:
      • HTTP → Porta 80
      • HTTPS → Porta 443
      • SMTP → Porta 25
  3. Cabeçalhos da Camada de Transporte
    • Contêm informações para identificar portas, controle de erro e ordem dos dados.

Serviços da Camada de Transporte

  1. Protocolos principais:
    • UDP (User Datagram Protocol)
      • Não confiável (sem garantia de entrega).
      • Mais rápido, usado em streaming, jogos online e VoIP.
    • TCP (Transmission Control Protocol)
      • Confiável (garante entrega, ordem e integridade dos dados).
      • Usado em navegação, e-mails, downloads.
  2. Modelo de serviço TCP
    • Orientado à conexão: antes de transferir dados, precisa do "handshake" (3 vias).
    • Controle de fluxo e congestionamento: ajusta velocidade de envio.
    • Retransmissão em caso de perda de pacotes.
  3. Mecanismos de confiabilidade
    • Go-Back-N: retransmite todos os pacotes a partir de um erro.
    • Selective Repeat: retransmite apenas os pacotes perdidos.
  4. Tipos de portas
    • Bem conhecidas (0-1023): usadas por serviços padrão (HTTP, DNS, SMTP).
    • Registradas (1024-49151): usadas por aplicações específicas.
    • Dinâmicas/Privadas (49152-65535): usadas temporariamente pelos clientes.
  • Camada de Aplicação → Interação com o usuário (HTTP, DNS, SMTP, POP3, IMAP).
  • Camada de Transporte → Comunicação confiável ou não confiável entre processos (TCP x UDP).
