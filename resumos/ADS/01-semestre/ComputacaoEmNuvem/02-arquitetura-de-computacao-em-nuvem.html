Arquitetura de Computação em Nuvem

O que é?

A arquitetura de computação em nuvem é a forma como os componentes da nuvem (servidores, redes, armazenamento, software) são organizados e se comunicam para entregar serviços ao usuário final.

Analogia Simples: Imagine a nuvem como um restaurante

  • Cozinha → onde tudo é preparado (infraestrutura e servidores).
  • Garçons → levam os pedidos e entregam (redes e APIs).
  • Menu → opções que você pode escolher (serviços SaaS, PaaS, IaaS).

Características Básicas da Arquitetura

  1. Elasticidade: aumenta ou diminui recursos automaticamente.
  2. Escalabilidade: expande a capacidade conforme a demanda.
  3. Segurança: protege dados e acessos.
  4. Automação: recursos e serviços configurados automaticamente.
  5. Alta disponibilidade: serviços sempre ativos e redundantes.

Modelos de Camadas

→ A arquitetura é geralmente organizada em três camadas:

  1. Camada de Infraestrutura (IaaS)
    • Inclui servidores físicos, máquinas virtuais, redes e armazenamento.
    • Exemplo: AWS EC2, Google Compute Engine.
  2. Camada de Plataforma (PaaS)
    • Fornece ambiente para desenvolver e hospedar aplicações sem gerenciar a infraestrutura.
    • Exemplo: Heroku, Azure App Service.
  3. Camada de Aplicação (SaaS)
    • Software pronto acessado pela internet.
    • Exemplo: Gmail, Google Drive.

Componentes Principais

  • Front-end: interface que o usuário vê (navegador, app).
  • Back-end: servidores e serviços que processam dados.
  • Gerenciamento: monitora, aloca e otimiza recursos (painéis de controle e APIs).
  • Segurança: autenticação, criptografia, backups.
  • Armazenamento: bancos de dados e repositórios de arquivos.

Serviços de Arquitetura

  • Computação: processar aplicações (máquinas virtuais, containers).
  • Armazenamento: guardar dados (banco de dados, armazenamento de objetos).
  • Rede: conectar recursos (VPN, balanceadores de carga).
  • Monitoramento: acompanhar desempenho (CloudWatch, Azure Monitor).
  • Segurança: firewall, autenticação multifator, criptografia.

Para Fixar o Conteúdo:

    Você é contratado para criar um sistema de e-commerce que precisa:
  1. Ter alta disponibilidade e escalabilidade.
  2. Permitir desenvolvimento rápido sem se preocupar com servidores.
  3. Usar um banco de dados na nuvem.
    → Resposta:
  1. IaaS para infraestrutura (AWS EC2).
  2. PaaS para desenvolvimento (Heroku ou AWS Elastic Beanstalk).
  3. Serviço de banco de dados (AWS RDS ou Firebase).
