Ambiente de Computação em Nuvem – Google Cloud (GCP)

O que é Google Cloud?

O Google Cloud Platform (GCP) é a plataforma de nuvem do Google.

Ela oferece serviços de computação, armazenamento, banco de dados, rede, inteligência artificial e análise de dados, com foco em alta performance, segurança e escalabilidade global.

→ Exemplo prático:

  • Hospedar vídeos no Cloud Storage.
  • Criar aplicações escaláveis no App Engine.
  • Usar BigQuery para análise de grandes volumes de dados.

Diferenciais do Google Cloud

  • Infraestrutura global → mesma rede que suporta YouTube e Gmail.
  • Big Data e IA → destaque em análise de dados e machine learning.
  • Kubernetes → o Google criou o Kubernetes, e o GCP oferece o GKE (Google Kubernetes Engine) como serviço gerenciado.
  • Sustentabilidade → foco em energia limpa e operações sustentáveis.

Principais Serviços

  1. Computação
    • Compute Engine → máquinas virtuais (equivalente ao EC2 da AWS).
    • App Engine → PaaS para aplicativos web sem gerenciar servidores.
    • Kubernetes Engine (GKE) → cluster Kubernetes gerenciado.
    • Cloud Functions → funções serverless sob demanda.
      • # Criar VM no Compute Engine via CLI
gcloud compute instances create minha-vm --zone=us-central1-a --machine-type=e2-micro
  2. Armazenamento e Banco de Dados
    • Cloud Storage → armazenamento de objetos (equivalente ao S3).
    • Persistent Disk → armazenamento em bloco para VMs.
    • Firestore / Firebase DB → banco de dados NoSQL.
    • Cloud SQL → banco relacional gerenciado (MySQL, PostgreSQL).
  3. Rede
    • VPC (Virtual Private Cloud) → redes privadas virtuais.
    • Cloud Load Balancing → balanceamento de tráfego global.
    • Cloud CDN → distribuição de conteúdo.
  4. Big Data e IA
    • BigQuery → análise de grandes volumes de dados em segundos.
    • AI Platform → machine learning.
    • Vertex AI → IA generativa e modelos de ML.

Boas Práticas de Segurança e Operações

  • IAM (Identity and Access Management) → controle de identidades e permissões.
  • Criptografia padrão para dados em trânsito e em repouso.
  • Monitoramento com Cloud Monitoring e Cloud Logging.
  • Política de menor privilégio para usuários e serviços.
  • Backups e redundância geográfica para alta disponibilidade.

Aplicações Práticas no Google Cloud

  • Hospedagem de sites estáticos: Cloud Storage + Cloud CDN.
  • Aplicações dinâmicas: App Engine ou GKE.
  • Machine Learning: treinar modelos com Vertex AI.
  • Análise de dados: BigQuery para relatórios em tempo real.

Para Fixar o Conteúdo:

    Você precisa desenvolver um sistema para análise de grandes volumes de dados de uma empresa. Indique:
  1. Qual serviço usar para análise de dados massivos?
  2. Qual recurso usar para hospedar a aplicação web que fará consultas?
  3. Qual serviço usar para armazenar arquivos CSV com dados brutos?
  4. Cite uma prática de segurança para proteger o acesso dos analistas ao sistema.
    → Resposta:
  1. BigQuery.
  2. App Engine ou GKE.
  3. Cloud Storage.
  4. Controle de acesso com IAM + autenticação multifator.
