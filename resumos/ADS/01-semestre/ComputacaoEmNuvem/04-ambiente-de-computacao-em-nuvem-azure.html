Ambiente de Computação em Nuvem – Azure

O que é Azure?

O Microsoft Azure é a plataforma de computação em nuvem da Microsoft, que oferece serviços como armazenamento, rede, bancos de dados e aplicativos, permitindo criar, testar, implantar e gerenciar aplicações na nuvem.

→ Exemplo prático:

  • Hospedar um site ou aplicativo web sem precisar ter servidores físicos próprios.
  • Criar bancos de dados na nuvem acessíveis de qualquer lugar.

Modelos e Tipos de Nuvem no Azure

  1. Modelos de Serviço
    • IaaS: máquinas virtuais, redes e armazenamento (ex: Azure Virtual Machines).
    • PaaS: ambiente para desenvolver e hospedar aplicativos sem gerenciar servidores (ex: Azure App Service).
  2. Tipos de Implantação
    • Nuvem Pública: recursos compartilhados (ex: serviços padrão do Azure).
    • Nuvem Privada: infraestrutura dedicada à empresa.
    • Nuvem Híbrida: mistura de pública e privada, permitindo integração de ambientes locais e na nuvem.

Componentes de Arquitetura e Serviços do Azure

  • Compute: máquinas virtuais, containers, funções serverless.
  • Storage: armazenamento de arquivos, blobs, filas e bancos de dados.
  • Network: redes virtuais, VPN, balanceadores de carga.
  • Database: SQL Database, Cosmos DB.
  • Analytics e AI: processamento de dados, Machine Learning e análise preditiva.
  • Identity & Security: Azure Active Directory, autenticação multifator, firewalls.

→ Exemplo prático de criar máquina virtual via Azure CLI:

# Criar VM básica no Azure
az vm create \
  --resource-group MeuGrupo \
  --name MinhaVM \
  --image UbuntuLTS \
  --admin-username azureuser \
  --generate-ssh-keys

Gerenciamento, Governança e Monitoramento

  • Azure Portal: interface gráfica para criar, configurar e monitorar recursos.
  • Azure Resource Manager (ARM): organiza recursos em grupos e permite automação via templates.
  • Monitoramento: Azure Monitor e Log Analytics ajudam a acompanhar desempenho e detectar falhas.
  • Políticas e Governança: controlam permissões, conformidade e limites de uso de recursos.

Aplicações Práticas

  • Migração de Sistemas: mover sistemas legados para Azure para reduzir custo e melhorar escalabilidade.
  • Hospedagem de Aplicativos Web: criar aplicativos que podem escalar automaticamente conforme a demanda.
  • Backup e Recuperação de Desastres: proteger dados críticos usando armazenamento na nuvem.
