Ambiente de Computação em nuvem - AWS

O que é AWS?

A Amazon Web Services (AWS) é a plataforma de computação em nuvem da Amazon.

Oferece serviços de computação, armazenamento, banco de dados, rede, inteligência artificial e muito mais, permitindo construir desde pequenos sites até sistemas globais de grande escala.

→ Exemplo prático:

  • Hospedar um site estático no S3.
  • Criar uma máquina virtual EC2 para rodar uma aplicação web.
  • Usar EFS para armazenamento compartilhado entre servidores.

Principais Soluções de Computação

  • Amazon EC2 (Elastic Compute Cloud): máquinas virtuais configuráveis sob demanda.
  • Auto Scaling: aumenta ou diminui instâncias conforme a carga.
  • Elastic Beanstalk: plataforma para implantar e gerenciar aplicativos sem se preocupar com infraestrutura (PaaS).

→ Criar uma VM no EC2 via AWS CLI:

aws ec2 run-instances \
  --image-id ami-123456 \
  --count 1 \
  --instance-type t2.micro \
  --key-name MinhaChave \
  --security-groups MeuGrupoSeguranca

Opções de Armazenamento na AWS

  1. EBS (Elastic Block Store)
    • Disco virtual anexado a instâncias EC2.
    • Ideal para sistemas operacionais e bancos de dados.
  2. S3 (Simple Storage Service)
    • Armazenamento de objetos (arquivos, imagens, vídeos).
    • Muito usado para sites estáticos e backups.
  3. EFS (Elastic File System)
    • Sistema de arquivos escalável e compartilhado.
    • Pode ser acessado por várias instâncias EC2 ao mesmo tempo.

Rede na AWS

  • VPC (Virtual Private Cloud): rede virtual isolada para seus recursos na nuvem.
  • Sub-redes: dividem a VPC em partes menores (pública e privada).
  • Internet Gateway: permite que recursos na VPC acessem a internet.
  • Route Tables (Tabelas de Rotas): definem o tráfego dentro da rede.
  • Elastic IPs: endereços IP fixos que podem ser atribuídos a instâncias.

→ Criar uma VPC via CLI:

aws ec2 create-vpc --cidr-block 10.0.0.0/16

Aplicações Web na AWS

  1. Aplicações Estáticas:
    • Hospedar páginas HTML, CSS e JS no Amazon S3.
    • Integrar com CloudFront para distribuir globalmente com baixa latência.
  2. Aplicações Dinâmicas:
    • Usar EC2 ou Elastic Beanstalk para rodar aplicações que processam dados no servidor.
    • Banco de dados pode ser hospedado no Amazon RDS (SQL) ou DynamoDB (NoSQL).

Resumo rápido para lembrar

  • Azure → melhor integração para empresas Microsoft.
  • AWS → mais serviços e pioneira em nuvem.
  • Ambas oferecem IaaS, PaaS, SaaS, escalabilidade, segurança e bancos de dados.

  1. S3 (site estático).
